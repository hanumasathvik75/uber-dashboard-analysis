# Uber Ride Booking Data Analysis

## Project Overview
This project analyzes Uber ride booking data to understand ride demand, cancellations, payment preferences, and vehicle type performance. Data cleaning was performed using Python (Pandas) and the insights were visualized using an interactive Power BI dashboard.

---

## Tools Used
- Python
- Pandas
- Power BI

---

## Data Cleaning Steps
The dataset was cleaned using Python before building the dashboard.

Cleaning tasks performed:
- Removed duplicate records
- Handled missing values (NA / NaN)
- Standardized booking status and payment methods
- Corrected inconsistent values
- Dropped unnecessary columns

---

## Dashboard Features
A **5-page Power BI dashboard** was created including:

- KPI cards for total bookings, completed rides, and cancelled rides
- Monthly booking trend analysis
- Vehicle type performance comparison
- Payment method distribution
- Cancellation analysis
- Driver and customer ratings overview

---

## Key Insights

### Total Demand
- The dataset contains around **150,000 ride bookings** including completed, cancelled, and incomplete rides.

### Monthly Trends
- Ride bookings fluctuate across months, showing varying demand patterns.

### Cancellations
- Most ride cancellations are initiated by **drivers**.

### Payment Methods
- **UPI is the most preferred payment method**, while card and bank payments are the least used.

### Vehicle Type Performance
- **Auto rides dominate the platform** with the highest number of bookings and travel distance.

### Ratings
- Driver and customer ratings are **fairly consistent across vehicle types**, indicating stable service quality.
